Introduction

This Shiny app is a course project for the Cousera Data Science Specialization - Developing Data Products course. This app estimates the average home price in Boston area in 1978, given its location, size and history. The prediction is based on a linear regression model using data from Harrison et al. [1].

 

Data & Model

The 1979 Boston housing data was obtained from UCI Machine Learning Repository [2]. For this work, we only focused on the following features to predict the median value of home price (MEDV):

  • TOWN: the town the house was located.
  • CHAS: wheter the house was near the Charles River.
  • RM: the number of rooms per home.
  • AGE: proportion of houses built prior to 1940.

Additionally county name and geocode information was added based on public available sources [3, 4].

The MEDV was fit with TOWN, CHAS, RM and AGE with a linear regression model using R programming language [5]. The reason we choose a linear model is mainly for performance consideration, because the model is trained when a user launches the app.

 

User Interface

The app adopted the default layout of Shiny app pacakge with the interface divided into a left sidebar panel and a main panel. The sidebar contains options to allow user to choose the location (first narrowed by county and then by town), the rooms and the age of the house. If a user does not make a choice, a set of default values will be used instead.

Once the parameters are specified, the app will compute the house price based on our linear model and present the data in the main panel. The data presentation include three sections:

  • A Google map showing the location of the house (approximated by the center of the town).
  • A brief description of the house.
  • The estimated price of the house in 1978.

A user may choose to examine many locations and freely adjust other parameters, the model will re-compute the price estimate and refresh the main panel content reactively.
